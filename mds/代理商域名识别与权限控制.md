# 代理商域名识别与权限控制体系

## 一、域名识别代理商 ID

### 核心流程

**入口中间件：`middleware/tenant.go` — `TenantMiddleware()`**

1. 从请求中提取 `c.Request.Host`（去掉端口）
2. 通过 `getAgentIdFromDomain()` 用两种方式匹配代理商：
   - **自定义域名精确匹配**：查询 `agents.domain` 字段
   - **子域名 slug 匹配**：提取第一段子域名（如 `agent1.example.com` → `agent1`），匹配 `agents.slug` 字段
   - 跳过 `www`、`localhost`、`127` 等通用子域名
3. 将 `AgentId` 写入请求上下文（主站默认为 0）
4. 验证代理商状态为 `active`

### Agent 模型关键字段

| 字段 | 说明 |
|------|------|
| `Id` | 代理商主键 |
| `Slug` | 子域名标识（唯一索引） |
| `Domain` | 自定义域名（唯一索引） |
| `Status` | 状态：`active` / `suspended` / `inactive` |

### 安全隔离

| 环节 | 文件 | 逻辑 |
|------|------|------|
| **登录校验** | `controller/user.go` | 验证 `user.AgentId == domainAgentId`，不匹配则拒绝："账号不属于当前站点" |
| **请求校验** | `middleware/auth.go` | 每次请求检查 session `agent_id` 与域名解析的 `domainAgentId` 一致 |
| **防伪造** | — | 不接受 header 伪造，域名是唯一权威来源 |

### 关键文件

| 文件 | 作用 |
|------|------|
| `middleware/tenant.go` | 域名 → AgentId 解析 |
| `model/agent.go` | Agent 模型定义 |
| `model/agent_methods.go` | `GetAgentByDomain()`、`GetAgentBySlug()` |
| `controller/user.go` | 登录时租户边界校验 |
| `middleware/auth.go` | 请求级租户隔离 |

## 二、权限控制体系（RBAC）

### 整体架构

系统基于 **Casbin RBAC** 实现三层控制：

| 层级 | 控制什么 | 权限码格式 | 示例 |
|------|----------|-----------|------|
| **菜单可见性** | 侧边栏/标签页是否显示 | `menu:{featureKey}` | `menu:user` |
| **API 访问** | 接口是否允许调用 | `{namespace}:{resource}:{action}` | `tenant:user:view` |
| **租户隔离** | 数据只能看自己代理商的 | `agent_id` 上下文 | — |

### 权限码规范

```
API 权限：{namespace}:{resource}:{action}
  - tenant:user:view        查看租户用户
  - tenant:user:create       创建用户
  - platform:channel:view    查看渠道（系统级）
  - platform:*               超级管理员通配

菜单权限：menu:{featureKey}
  - menu:user                用户管理菜单
  - menu:channel             渠道管理菜单

通配符：
  - platform:*               匹配所有权限
  - tenant:user:*            匹配用户相关所有操作
```

### 角色层级

```
root (level 7)          — 超级管理员
├── admin (level 6)     — 管理员
│   ├── sales (level 5)
│   ├── operator (level 4)
│   ├── finance (level 3)
│   └── purchase (level 2)
└── user (level 1)      — 普通用户
    └── guest (level 0)
```

父角色自动继承子角色的所有权限。

### 数据库表结构

| 表 | 用途 |
|----|------|
| `permissions` | 权限码定义（perm_type: 2=菜单, 4=API） |
| `permission_modules` | 功能模块分组 |
| `permission_features` | 功能定义，自动派生权限 |
| `roles` | 角色定义（含层级） |
| `role_permissions` | 角色-权限绑定 |
| `user_roles` | 用户-角色绑定（按代理商） |
| `casbin_rule` | Casbin RBAC 策略存储 |

### 权限检查流程

```
请求 → UserAuth() 验证登录/Token
     → RequirePerm("x:y:z") 检查权限
       → Casbin Enforce("user:123", "agentId", "x:y", "z")
       → 通过 → 执行业务逻辑
       → 拒绝 → HTTP 403 "缺少权限: x:y:z"
```

## 三、开发新页面（需写代码的部分）

> 权限配置（创建模块、功能、角色、分配权限）全部通过管理后台 UI 完成，见第五章。
> 本章只涉及**必须写代码**的两个环节：后端路由 + 前端页面。

### 第 1 步：后端路由加权限中间件

假设已在 `/console/features` 创建了 4 个 Feature，权限码已自动派生。

```go
// router/api-router.go
monetaryRoute := apiRouter.Group("/monetary")
monetaryRoute.Use(UserAuth())
{
    // 概览
    monetaryRoute.GET("/overview", RequirePerm("tenant:monetary_overview:view"))
    // 记录
    monetaryRoute.GET("/records", RequirePerm("tenant:monetary_records:view"))
    monetaryRoute.POST("/records", RequirePerm("tenant:monetary_records:create"))
    // 设置
    monetaryRoute.GET("/settings", RequirePerm("tenant:monetary_settings:view"))
    monetaryRoute.PUT("/settings", RequirePerm("tenant:monetary_settings:update"))
    // 审计
    monetaryRoute.GET("/audit", RequirePerm("tenant:monetary_audit:view"))
}
```

### 第 2 步：前端标签页按权限控制显示

```jsx
import { hasPermission } from '@/helpers/utils';
import { Tabs, TabPane } from '@douyinfe/semi-ui';

// 定义标签页及其权限要求
const allTabs = [
  { key: 'overview',  label: '概览', perm: 'tenant:monetary_overview:view' },
  { key: 'records',   label: '记录', perm: 'tenant:monetary_records:view' },
  { key: 'settings',  label: '设置', perm: 'tenant:monetary_settings:update' },
  { key: 'audit',     label: '审计', perm: 'tenant:monetary_audit:view' },
];

// 过滤出有权限的标签页
const visibleTabs = allTabs.filter(tab => hasPermission(tab.perm));

<Tabs>
  {visibleTabs.map(tab => (
    <TabPane key={tab.key} tab={tab.label}>
      {/* 标签页内容 */}
    </TabPane>
  ))}
</Tabs>
```

## 四、总结

```
代理商自定义域名/子域名
       ↓
TenantMiddleware 解析 → AgentId
       ↓
UserAuth 验证登录 → UserId + AgentId
       ↓
RequirePerm 检查权限 → Casbin RBAC
       ↓
前端 hasPermission() → 标签页/按钮显示控制
```

**核心原则：后端 `RequirePerm()` 保安全，前端 `hasPermission()` 保体验，两层配合。**

## 五、前端管理界面操作指南

以下是通过管理后台 UI 完成"新页面 + 多标签页权限控制"的完整操作流程，无需写代码即可完成权限配置。

### 5.1 功能管理页面（`/console/features`）

> 需要 `platform:*` 权限（超级管理员）

这是权限体系的起点，所有权限码都从这里的配置自动派生。

#### 操作一：创建模块（Module）

模块是功能的分组容器，对应侧边栏的一级菜单分类。

1. 进入 `/console/features` 页面
2. 点击左上角 **"新增模块"** 按钮
3. 弹出「新增模块」对话框，填写：

| 字段 | 说明 | 示例 |
|------|------|------|
| 模块标识 | 唯一英文标识，创建后不可改 | `finance` |
| 模块名称 | 中文显示名称 | `财务管理` |
| 图标 (Lucide Icon) | 从 Lucide 图标库搜索选择 | `Wallet` |
| 排序号 | 数字越小越靠前 | `10` |

4. 点击确认保存
5. 页面顶部的 Tab 栏会新增一个 **"财务管理"** 标签

> 模块标签上可以直接点击排序号数字修改排序，也可以点铅笔图标编辑模块信息。

#### 操作二：创建功能（Feature）

功能对应具体的页面/标签页，系统会根据 Actions 自动生成权限码。

1. 在顶部 Tab 栏切换到目标模块（如 **"财务管理"**）
2. 点击 Tab 栏右侧的 **"新增功能"** 按钮（蓝色实心按钮）
3. 弹出「新增功能」对话框，填写：

| 字段 | 说明 | 示例 |
|------|------|------|
| 功能标识 (FeatureKey) | 唯一英文标识，**创建后不可修改** | `monetary_overview` |
| 功能名称 | 中文显示名称 | `资金概览` |
| 所属模块 | 下拉选择已创建的模块 | `finance (财务管理)` |
| 命名空间 (Namespace) | `tenant`（代理商级）或 `platform`（平台级） | `tenant` |
| 前端路由 | 该功能对应的前端页面路径 | `/console/monetary` |
| 图标 (Lucide Icon) | 搜索选择图标 | `BarChart` |
| 排序号 | 功能在模块内的排序 | `1` |
| 功能动作 | 勾选需要的操作（见下方说明） | 勾选 `菜单入口` + `查看` |
| 自定义动作 | 内置 5 个不够用时，逗号分隔追加 | `export, assign` |
| API Code 覆盖 | 可选，覆盖自动生成的权限码 | 一般留空 |

**功能动作（Actions）说明：**

弹窗中有 5 个内置复选框：

| 复选框 | 含义 | 自动生成的权限码 |
|--------|------|-----------------|
| ☑ 菜单入口 | 在侧边栏/标签页中可见 | `menu:{featureKey}` |
| ☑ 查看 | 可调用查看/列表接口 | `{namespace}:{featureKey}:view` |
| ☐ 创建 | 可调用创建接口 | `{namespace}:{featureKey}:create` |
| ☐ 编辑 | 可调用修改接口 | `{namespace}:{featureKey}:update` |
| ☐ 删除 | 可调用删除接口 | `{namespace}:{featureKey}:delete` |

默认新增功能时已预选 **菜单入口 + 查看**。

4. 点击确认保存，权限码立即自动派生

│                                         │

│  ▶ 系统设置（已折叠）                       │
│                                         │

**智能联动规则：**
- 勾选 **"菜单"** → 自动勾选 **"查看"**（菜单可见必须能查看）

- **看不到** "资金审计" 标签页
- 交易记录可以创建新记录，资金设置可以修改，但概览只能查看

### 5.5 完整操作流程（端到端示例）

**目标：** 新建一个"资金管理"页面，含 4 个标签页，不同代理商的角色能看到不同标签。

```
步骤 1：功能管理 (/console/features)
  │
  ├─ 创建模块 "finance"（如果不存在）
  │
  ├─ 创建 Feature: monetary_overview  (actions: menu, read)
  ├─ 创建 Feature: monetary_records   (actions: menu, read, create)
  ├─ 创建 Feature: monetary_settings  (actions: menu, read, update)
  └─ 创建 Feature: monetary_audit     (actions: menu, read, delete)
       │
       ▼ 系统自动派生 20 个权限码

步骤 2：角色管理 (/console/role)
  │
  ├─ 创建角色 "finance_admin"
  │   └─ 权限抽屉：勾选所有 4 个功能的全部操作 ✓
  │
  ├─ 创建角色 "finance_viewer"
  │   └─ 权限抽屉：只勾选 overview 和 records 的菜单+查看 ✓
  │
  └─ 创建角色 "finance_auditor"
      └─ 权限抽屉：勾选 overview + audit 的菜单+查看+删除 ✓
       │
       ▼

步骤 3：用户管理 (/console/user)
  │
  ├─ 代理商 A 的管理员 → 分配 finance_admin（看到全部 4 个标签）
  ├─ 代理商 B 的员工   → 分配 finance_viewer（只看到概览+记录）
  └─ 审计人员          → 分配 finance_auditor（只看到概览+审计）
       │
       ▼

步骤 4（开发）：前端页面 + 后端路由
  │
  ├─ 前端：用 hasPermission() 过滤标签页
  └─ 后端：路由加 RequirePerm() 中间件
```

### 5.6 前端权限检查函数速查
