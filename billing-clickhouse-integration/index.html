WORKER Analytics Worker — Go 独立服务
技术选型
Go 独立进程，和 Gin 分开部署
ClickHouse 客户端: clickhouse-go (官方推荐)
连接方式: native protocol (9000 端口)
写入策略
优先自己攒批次: 每 1000 条 或 每 2 秒 flush
备选: async_insert=1, wait_for_async_insert=1
避免大量小批次同步插入 (too many parts)
FLOW Worker 处理流程
后续演进: Worker -> Kafka -> Consumer
等需要多下游消费 / 事件回放 / consumer lag 监控时，把 outbox -> worker 升级为 outbox -> Kafka -> consumer。 ClickHouse 官方支持 Kafka table engine 直接消费。当前阶段没必要一步上 Kafka。

CREATE TABLE fact_request_settlement ( ...columns... ) ENGINE = MergeTree() PARTITION BY toYYYYMM(biz_date) ORDER BY (biz_date, agent_id, model, request_id) TTL biz_date + INTERVAL 1 YEAR SETTINGS index_granularity = 8192
MV agg_minute — 分钟聚合 (Materialized View)
字段CH 类型说明
minuteDateTime分钟时间戳 (toStartOfMinute)
agent_idUInt32代理商 ID
modelLowCardinality(String)模型名
request_countAggregateFunction(count)请求数
sum_tokensAggregateFunction(sum, UInt64)token 总量
sum_user_amtAggregateFunction(sum, Int64)用户收入
sum_agent_amtAggregateFunction(sum, Int64)代理商结算
sum_platform_costAggregateFunction(sum, Int64)平台成本
sum_agent_profitAggregateFunction(sum, Int64)代理商利润
sum_platform_profitAggregateFunction(sum, Int64)平台利润
CREATE MATERIALIZED VIEW mv_agg_minute TO agg_minute AS SELECT toStartOfMinute(finished_at) AS minute, agent_id, model, countState() AS request_count, sumState(total_tokens) AS sum_tokens, sumState(actual_user_amt) AS sum_user_amt, sumState(actual_agent_amt) AS sum_agent_amt, sumState(actual_platform_cost) AS sum_platform_cost, sumState(agent_profit) AS sum_agent_profit, sumState(platform_profit) AS sum_platform_profit FROM fact_request_settlement GROUP BY minute, agent_id, model
MV agg_daily — 日聚合 (Materialized View)
字段CH 类型说明
dateDate日期
agent_idUInt32代理商
modelLowCardinality(String)模型
provider_idUInt32供应商
request_count / sum_tokens / sum_*_amt / sum_*_profitAggregateFunction(...)同分钟表
avg_latencyAggregateFunction(avg, UInt32)平均延迟
CREATE MATERIALIZED VIEW mv_agg_daily TO agg_daily AS SELECT biz_date AS date, agent_id, model, provider_id, countState() AS request_count, sumState(total_tokens) AS sum_tokens, sumState(actual_user_amt) AS sum_user_amt, -- ... same pattern ... avgState(latency_ms) AS avg_latency FROM fact_request_settlement GROUP BY date, agent_id, model, provider_id
物化视图工作原理
数据插入 fact_request_settlement 时，ClickHouse 自动把聚合结果写入 agg_minute 和 agg_daily。 计算成本从查询阶段转移到插入阶段。Gin 报表直接查聚合表，不用扫明细。
STEP 最小改造落地顺序
STEP 1
PG 新增两张表
在现有 BillingSession.Settle 事务里追加写入
  • 新建 request_settlement
  • 新建 event_outbox
  • 改 Settle 事务：写结算单 + outbox
  • 保留 logs 写入不变（兼容）
STEP 2
Go Analytics Worker
独立进程，轮询 outbox，批量写 CH
  • Go 独立服务
  • 用 clickhouse-go
  • 每 1-2 秒扫一次 outbox
  • 攒批 1000 条 flush
  • 写入成功标 SENT
STEP 3
ClickHouse 表 + 物化视图
建明细表 + 自动聚合
  • 部署 ClickHouse
  • 建 fact_request_settlement
  • 建 agg_minute MV
  • 建 agg_daily MV
  • 灌入历史数据 (从 PG logs 同步)
STEP 4
Gin 报表接口
新增 / 改造查询接口
  • 新增报表接口查 ClickHouse
  • 交易查询仍走 PG
  • PG billing_daily_stats 降级备份
  • 30min cron 聚合保留做兜底
